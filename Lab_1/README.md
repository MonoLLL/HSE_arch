## Перечень заинтересованных лиц (стейкхолдеров)
| № | Стейкхолдер | Краткое описание |
|---|-------------|------------------|
| 1 | Студент | Основной пользователь системы. Получает объявления, создаёт обращения, отслеживает статусы и получает ответы. |
| 2 | Сотрудник учебного офиса | Обрабатывает обращения студентов, отвечает на вопросы, публикует объявления. |
| 3 | Технический администратор системы | Отвечает за поддержку системы, обновления, управление пользователями, настройку категорий обращений и структуру сайта. |

## Функциональные требования

### Функции для студента
- Просмотр объявлений учебного офиса
- Через поиск найти нужный ответ на вопрос

### Функции для сотрудника учебного офиса 
- Создание объявлений
- Настройка категорий обращений

### Функции для технического администратора
- Управление ролями 
- Управление структурой сайта.  
- Просмотр логов и мониторинг работы системы
- Установка обновлений и исправлений

## 4. Диаграмма вариантов использования



## 5. Сделанные предположения
 
- Объявления отображаются всем студентам.  
- Сотрудники учебного офиса не имеют доступа к техническим настройкам.  
- Технический администратор - единственный, кто может менять структуру сайта.  
- Система используется внутри учебной организации.

---

## 6. Нефункциональные требования

### **Доступность**
Система должна работать не менее 99% времени, так как используется ежедневно.

### **Безопасность**
- Ограничение доступа к роли сотрудника учебного офиса.  

### **Удобство использования**
Интерфейс должен позволять студенту быстро получить нужную информацию.
